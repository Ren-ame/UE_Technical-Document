Introduction

JavaScript is a lightweight, interpreted, or just-in-time compiled programming language with first-class functions. While it is most well-known as the scripting language for Web pages, many non-browser environments also use it, such as Node.js, Apache CouchDB, and Adobe Acrobat.

JavaScript is a prototype-based, multi-paradigm, single-threaded, dynamic language, supporting object-oriented, imperative, and declarative (e.g. functional programming) styles. JavaScript's dynamic capabilities include runtime object construction, variable parameter lists, function variables, dynamic script creation (via eval), object introspection, and source-code recovery.

The standard for JavaScript is ECMAScript. As of 2012, all modern browsers fully support ECMAScript 5.1. Older browsers support at least ECMAScript 3. On June 17, 2015, ECMA International published the sixth major version of ECMAScript, which is officially called ECMAScript 2015 but is still more commonly referred to as ES6 or ES2015.

Do not confuse JavaScript with the Java programming language. Both "Java" and "JavaScript" are trademarks or registered trademarks of Oracle in the U.S. and other countries. However, the two programming languages have very different syntax, semantics, and use.

To get started with JavaScript, you only need a modern web browser. Open your browser's developer console to start writing and testing JavaScript code immediately.

Variables and Data Types

JavaScript has three keywords for declaring variables: var, let, and const. The let and const keywords were introduced in ES6 and are now the preferred way to declare variables.

JavaScript is a dynamically typed language, meaning you do not need to specify the data type of a variable when declaring it. The type is determined automatically at runtime. The main data types in JavaScript are:

  • Number – integers and floating-point numbers
  • String – text values
  • Boolean – true or false
  • null – intentional absence of a value
  • undefined – a variable that has not been assigned a value
  • Object – collections of key/value pairs
  • Symbol – unique identifiers (ES6)

Here are examples of declaring variables with different data types:

let name = "Alice"; // String
let age = 30; // Number
let isActive = true; // Boolean
let score = null; // null
let address; // undefined
const PI = 3.14159; // Number (constant)

Use typeof to check the type of a variable at runtime. For example, typeof name returns "string" and typeof age returns "number".

Functions

Functions are one of the fundamental building blocks in JavaScript. A function is a reusable block of code designed to perform a particular task. Functions are executed when they are called (invoked).

JavaScript provides several ways to define functions. The traditional way uses the function keyword, while ES6 introduced arrow functions as a more concise syntax.

// Function declaration
function greet(name) {
  return "Hello, " + name + "!";
}

// Function expression
const greet = function(name) {
  return "Hello, " + name + "!";
};

// Arrow function (ES6)
const greet = (name) => "Hello, " + name + "!";

Functions can accept parameters and return values. If no return statement is provided, the function returns undefined by default.

JavaScript also supports higher-order functions — functions that take other functions as arguments or return them. This is a key concept in functional programming with JavaScript, and is widely used with array methods like map, filter, and reduce.

const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map(n => n * 2);
// doubled: [2, 4, 6, 8, 10]

const evens = numbers.filter(n => n % 2 === 0);
// evens: [2, 4]
Control Flow

Control flow refers to the order in which individual statements, instructions, or function calls are executed. JavaScript provides several control flow structures including conditionals and loops.

The if...else statement executes a block of code if a specified condition is truthy. If the condition is falsy, another block of code (the else block) can be executed.

let temperature = 25;

if (temperature > 30) {
  console.log("It's hot outside!");
} else if (temperature > 20) {
  console.log("It's a nice day!");
} else {
  console.log("It's a bit cold.");
}

JavaScript provides several looping structures: for, while, do...while, for...of, and for...in. Choose the loop type that best matches your use case.

The switch statement evaluates an expression and executes the case that matches. It is often a cleaner alternative to multiple if...else if chains when comparing the same variable against many values.

for (let i = 0; i < 5; i++) {
  console.log(i);
}

const fruits = ["apple", "banana", "cherry"];
for (const fruit of fruits) {
  console.log(fruit);
}
Objects and Arrays

Objects are collections of properties, where each property is a key-value pair. Objects are used to model real-world entities and are the foundation of object-oriented programming in JavaScript.

Arrays are ordered collections of values. They can hold values of any type, including numbers, strings, objects, and even other arrays. Arrays are zero-indexed, meaning the first element is at index 0.

// Object literal
const person = {
  name: "Alice",
  age: 30,
  greet() {
    return `Hi, I'm ${this.name}`;
  }
};

console.log(person.name); // "Alice"
console.log(person.greet()); // "Hi, I'm Alice"

JavaScript arrays come with many built-in methods such as push, pop, shift, unshift, splice, slice, map, filter, and reduce. These methods make it easy to manipulate and transform data.

Destructuring assignment is a powerful ES6 feature that allows you to unpack values from arrays or properties from objects into distinct variables, making your code cleaner and more readable.

// Array destructuring
const [first, second, ...rest] = [1, 2, 3, 4, 5];

// Object destructuring
const { name, age } = person;
console.log(name); // "Alice"
console.log(age); // 30
ES6 Features

ECMAScript 2015 (ES6) introduced many new features that modernized JavaScript. These features make the language more expressive, reduce boilerplate, and improve readability. Key ES6 features include arrow functions, template literals, destructuring, default parameters, rest/spread operators, classes, modules, and Promises.

Template literals allow you to embed expressions inside strings using backtick syntax. They support multi-line strings and expression interpolation.

const name = "World";
const greeting = `Hello, ${name}!`;
console.log(greeting); // "Hello, World!"

// Multi-line string
const message = `This is line one.
This is line two.`;

Classes in ES6 provide a cleaner, more familiar syntax for creating objects and handling inheritance. Under the hood, they still use JavaScript's prototype-based inheritance.

The spread (...) operator allows an iterable such as an array to be expanded in places where zero or more arguments or elements are expected. The rest parameter syntax allows a function to accept an indefinite number of arguments as an array.

class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    return `${this.name} makes a sound.`;
  }
}

class Dog extends Animal {
  speak() {
    return `${this.name} barks.`;
  }
}
Asynchronous JavaScript

JavaScript is single-threaded, meaning it can only execute one piece of code at a time. Asynchronous programming allows JavaScript to perform long-running tasks (like network requests or file reads) without blocking the main thread, keeping applications responsive.

Historically, callbacks were used to handle asynchronous operations. Promises, introduced in ES6, provide a cleaner way to work with async operations. The async/await syntax, introduced in ES2017, makes asynchronous code look and behave more like synchronous code.

// Using Promises
fetch("https://api.example.com/data")
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error(error));

The async keyword before a function makes it always return a Promise. The await keyword can only be used inside an async function and pauses execution until the Promise resolves.

Error handling in async/await is done using try...catch blocks, providing a familiar syntax for handling both synchronous and asynchronous errors in the same way.

async function getData() {
  try {
    const response = await fetch("https://api.example.com/data");
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error("Error:", error);
  }
}
